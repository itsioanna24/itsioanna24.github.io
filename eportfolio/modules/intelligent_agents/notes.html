Unit 1 – Introduction to Intelligent Agents

Lecturecast Notes

  • Agent definition: a system capable of undertaking independent actions on behalf of the user.
  • Agents perceive their environment through sensors (hardware or software-based) and act through effectors / actuators.
  • Autonomy exists on a spectrum — from fully automated to fully human-controlled.
  • Adjustable autonomy: control is handed back to the operator when the agent believes a human will make a better decision, when there is high uncertainty, when the decision might cause harm, or when the agent lacks the capability to decide alone.
  • Agent design begins with understanding the preconditions of the environment.

Examples of Agent-Based Systems

  • Control systems – e.g. a heating system
  • Software daemons – e.g. an email service (runs as a background process and wakes up to handle periodic service requests)
  • Reactive and functional systems – e.g. a software compiler

Unit 3 – Agent Architectures

Lecturecast Notes

Agent Architecture: the decomposition of an agent into a number of component modules and the way in which these modules interact with each other.

Three Types of Agents

  • Symbolic Reasoning Agents – a classical approach based on symbolic AI; agents are viewed as a type of knowledge-based system.
    • Problems: transduction problem; difficulty representing information symbolically.
  • Reactive Agents – respond directly to their environment without internal symbolic representations.
  • Hybrid Agents – combine deliberative and reactive layers.

Practical Reasoning

  • Practical reasoning: reasoning directed towards actions — figuring out what to do in a given situation.
  • Deliberation: deciding what outcomes to achieve; the output is intentions.
  • Means-end reasoning: deciding how to achieve those intentions; the output is a plan.

BDI Agents

  • Belief-Desire-Intention (BDI) agents are developed based on human practical reasoning.
  • Emergent behaviour: agent behaviours emerge from the interaction of rules with the environment.

Action Selection

The way an agent selects an action is defined by an action function.

Unit 5 – Agent Communication Languages

Lecturecast Notes

Semantics of Communication

The semantics of a request are made up of three components: s (the person making the request), h (the person the request is aimed at), and φ (the request itself). The postcondition is to make h aware of the desire to complete the request.

Agent Communication Languages (ACLs)

  • ACLs represent a standard format for the exchange of messages between agents.
  • KQML (Knowledge Query and Manipulation Language) was developed as part of the DARPA-funded Knowledge Sharing Effort.
  • KQML defines the outer language of acceptable communicative verbs — called performatives — including:
    • ask-if: allows an agent to ask if something is true.
    • perform: a request to perform an action.
    • tell: an agent’s response to an ask-if.
    • reply: gives an answer to a question.
  • KIF (Knowledge Interchange Format): a language for expressing message content or domain knowledge; based on first-order logic and can be used to express ontologies.

Ontology Alignment

  • Different contexts and requirements make modelling ontologies challenging.
  • Correspondence Inclusion Dialogue (CID): allows two agents to exchange knowledge about correspondences or alignments to agree on a mutually acceptable final alignment.
  • KQML limitation: lack of interoperability between implementations.

Unit 7 – Natural Language Processing

Lecturecast Notes

  • Natural Language Processing (NLP): a set of algorithms and processes by which machines understand and use human language.
  • Examples: Siri, Google Assistant, Alexa.
  • NLP is a non-trivial task.

Levels of Linguistic Analysis

Syntax → Semantics → Pragmatics

NLP Techniques

  • Statistical NLP: predicts the next word in a sequence given the words that precede it.
  • Word2Vec: learns the associations between words from a large text corpus. A word is assigned a vector value that captures both semantic and syntactic qualities, and vectors can indicate semantic similarity.
  • Constituency-Based Parse Trees: based on context-free grammar; sentences are subdivided into constituent phrases belonging to specific grammatical categories.

Choosing a Parser

  • Dependency parsing: more useful for downstream tasks like information extraction or question answering; works better in free word-order languages.
  • Constituency parsing: very useful when you want to consider the sub-phrases within a sentence.
  • Choice is situational.

Unit 9 – Deep Learning and Neural Networks

Lecturecast Notes

Adaptive Algorithms

An adaptive algorithm changes its behaviour at runtime, deciding the best way to act based on currently available information or a reward mechanism, rather than following a rigid plan.

Artificial Neural Networks (ANNs)

  • A class of machine learning models inspired by the brain and central nervous systems of animals.
  • Good for: classification tasks.
  • Made up of layers of nodes; each node consists of one or more artificial neurons.

ANN Training

  1. Create training data.
  2. Break the training data into features.
  3. Train the weights associated with each connection.
  4. Apply new input — the right neurons fire and make a decision.

Limitations of Neural Networks

  • Large volume of training data required.
  • Significant computational power needed.
  • Expert knowledge required.
  • Lacks a valid explanation for the classification.

Deep Learning vs Machine Learning

Machine LearningDeep Learning
Input → Feature Extraction → Classification → OutputInput → Feature Extraction + Classification → Output

Deep Learning Architecture

Layers: Input Layer → Hidden Layer → Output Layer

  • Forward propagation: progression of computation through the network.
  • Backpropagation: calculates errors in predictions and alters weights/biases by moving backward through layers, enabling the model to become increasingly accurate over time.

Convolutional Neural Networks (CNNs)

  • Can detect features and patterns within images, enabling object detection and recognition.
  • Examples: computer vision and image classification.
  • Convolutional layer: requires input data, a filter, and a feature map.
  • Pooling layer (downsampling): reduces the number of parameters.
    • Max Pooling: selects the pixel with the maximum value.
    • Average Pooling: calculates the average value within the receptive field.
  • Fully Connected Layer: all inputs from one layer are connected to every activation unit of the next layer.

Recurrent Neural Networks (RNNs)

  • Use sequential or time-series data.
  • Examples: NLP, speech recognition, image captioning.
  • Distinguish themselves by remembering prior inputs to influence current output.

Gradient Descent

An optimisation algorithm that finds the local minimum of a differentiable function; used to train neural networks by adjusting weights using derivatives of the network’s output error.

Unit 11 – Industry 4.0 and Intelligent Agents

Lecturecast Notes

Industry 4.0

A term capturing the significant evolution of processes and technology in the manufacturing world, driven by intelligent technologies including: Robotics, Internet of Things (IoT), 3D Printing, Additive Manufacturing, Digital Twinning, and Data Analytics.

  • Digital Twins: a virtual model designed to reflect the physical properties of a real-world object.
  • FANUC is one of the largest manufacturers of industrial robots in the world.

Types of Agents in Industry 4.0

  • Machine agents: perform manufacture or testing operations.
  • Conveying agents: move products and equipment around a facility.
  • Product agents: relate to the products being processed.
  • Supplementary agents: buffers for the products.

Cooperation of Smart Objects

Different components align their behaviours to achieve system-wide goals.

FinTech Applications

  • Credit risk assessment
  • Fraud detection
  • Financial modelling: using agent-based models to explain the behaviour of financial systems by simulating the actions of individual elements.

Generative AI

  • DALL-E: generates artwork based on text prompts using a technique called diffusion modelling, derived from GPT-3.
  • Diffusion models perform two sequential processes (forward and reverse diffusion).

Part 2 – Personal Development Action Plan

Where do I want to be by the end of this period? By the end of the next study period, I want to close the specific developmental gaps identified during the Intelligent Agents module — moving from rule-based to learning-enhanced agent design, strengthening my ability to enter new team environments proactively, and deepening my understanding of professional codes of conduct as they apply to autonomous AI systems.

What do I want / need to learn? What do I have to do to achieve this? Resources or support needed How will I measure success? Target dates
1. Extend from rule-based to learning-enhanced agent design
The individual presentation identified keyword-based filtering as the principal limitation of the system. The proposed future improvements — Scikit-learn classifiers, semantic similarity, and NLP embeddings — were identified but not implemented. I want to build practical fluency in integrating ML-based components into agent systems.		 Implement a text classification or relevance ranking model using Scikit-learn on the existing ArXiv pipeline; experiment with sentence embeddings (e.g. sentence-transformers) as a replacement for keyword scoring. Document the comparison of rule-based vs. learned relevance. Scikit-learn documentation; Hugging Face sentence-transformers library; existing BDI codebase on GitHub; academic literature on learning-enhanced BDI agents (da Silva et al., 2020; Qu et al., 2025). A working branch of the system uses a learned relevance model rather than keyword heuristics; I can document the performance difference and justify the choice of approach in writing. Within 3 months
2. Treat the team forming stage as an active contribution opportunity
The reflective piece identified that initial anxiety in Group E made me more reserved in the early meetings than I would typically be. I want to develop the confidence and practical habits to contribute proactively from the first team interaction rather than waiting for comfort to arrive.		 Before the next team project begins, re-read Tuckman (1965) and draft a personal checklist for the forming stage: introduce a clear contribution at Meeting 1, propose a team contract clause proactively, and set a personal target of raising at least one substantive idea in each of the first three meetings. Tuckman (1965); UoEO study skills resources on teamwork; reflection on Group E experience as a documented starting point. Peer evaluation feedback from the next team project reflects active early contribution; I do not describe anxiety or reserve as a factor in the forming stage in my next reflective piece. Start of next team-based assignment
3. Deepen technical depth in implementation documentation
The Group E tutor feedback noted that further detail on implementation mechanisms and runtime system behaviour would strengthen the technical depth of the proposal. I want to develop a personal standard for documenting runtime behaviour, including state transitions, edge cases, and logging evidence, in future technical deliverables.		 Review the Group E tutor feedback in detail; produce a supplementary technical document for the BDI system that captures runtime behaviour using the existing execution logs; apply the same standard to the next technical submission. Group E tutor feedback; existing system logs and unit test outputs; Sommerville (2016) on software documentation practices. Tutor feedback on the next technical submission no longer identifies runtime behaviour or implementation depth as a developmental point. End of next module
4. Formalise understanding of professional codes of conduct for autonomous systems
The Skills Matrix identifies this as currently Trained through engagement with Floridi et al. (2018) and lifecycle ethics. I want to move to a grounded understanding of specific professional codes relevant to the design and deployment of autonomous agent systems.		 Read the BCS Code of Conduct and the ACM Code of Ethics in full. Map specific clauses to the ethical themes engaged with in Collaborative Discussion 3 and the Deep Learning in Action activity, particularly on accountability, transparency, and data governance. BCS and ACM websites; Collaborative Discussion 3 summary post and portfolio entries as comparison points; Floridi et al. (2018) as a bridging framework. Able to cite specific clauses of the BCS or ACM code when discussing the ethical dimensions of agent system design in future module work or professional contexts. Within 2 months

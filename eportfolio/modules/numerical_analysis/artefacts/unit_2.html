Data Activity 2

1 Project Context

As part of my MSc coursework in Data Analysis and Programming, I worked with a real-world COVID-19 dataset containing daily state-level statistics from India. The aim of this formative activity was to import, clean, and analyse the dataset using R while demonstrating competency in key programming skills including file handling, data inspection, variable creation, frequency analysis, and simple descriptive statistics.

2 Data Import and Preparation

The dataset (covid19-dataset.csv) was imported into R using read.csv(). Several verification steps confirmed correct loading:

  • names(data) confirmed the expected variables: Sno, Date, State.UnionTerritory, ConfirmedIndianNational, ConfirmedForeignNational, Cured, Deaths.
  • nrow(data) verified that the dataset contains 270 observations.
  • head(data) allowed inspection of the first six rows to confirm correct loading.
Debugging note: During this process, newly created variables were initially returning NULL. This was traced back to R pointing to the wrong working directory. Updating it with setwd() resolved the issue, highlighting the importance of correct file path handling before any data operations.

Reflection

This activity reinforced several important aspects of real-world data analysis. Firstly, it highlighted the significance of correct file handling — seemingly small errors such as incorrect directory paths can lead to confusing outputs like NULL variables. Troubleshooting these issues improved my confidence working with R's file system and debugging tools, and underlined that a systematic setup (verifying the working directory before any data operations) should become a standard first step in any R workflow.

Secondly, creating the indicator variables recovery_flag and has_recovery deepened my understanding of how categorical and binary transformations can greatly enhance both interpretability and analytic flexibility. These transformations directly mirror common preprocessing steps in epidemiology and machine learning, where reducing a continuous or count variable to a binary indicator enables simpler, more communicable insights — and opens the door to techniques such as logistic regression or chi-square testing that require categorical inputs.

Finally, calculating proportions alongside raw counts provided a much clearer contextual picture. Knowing that only 20.37% of daily state reports included any recovered cases — and visualising this with a pie chart — gave a more immediate grasp of the recovery landscape in the early pandemic period than a raw table of 55 vs. 215 alone would have conveyed. This reinforced the importance of choosing the right summary representation for a given audience and analytical goal.

Overall, the activity strengthened my data preparation, problem-solving, and analytic reasoning skills in R, and demonstrated how even straightforward transformations and summaries can generate meaningful, interpretable insights from raw data — a core principle of applied data science.

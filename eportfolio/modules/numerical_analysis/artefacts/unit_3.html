Data Activity 3

This activity continued working with the COVID-19 India dataset, focusing on transforming numeric variables into binary indicators, categorical groups, and labelled factors — and then exploring the resulting distributions through frequency analysis and ranking.

Task 1 Creating the has_deaths Binary Variable

Raw death counts were converted into a binary indicator using ifelse(), assigning 1 where at least one death was recorded and 0 otherwise.

has_deathsCount
0 — No Deaths248

This task reinforced the importance of simplifying numeric variables into binary indicators for clearer analytical interpretation. Converting raw death counts into a 0/1 structure made it easier to conduct frequency analysis and compare reporting patterns. It also highlighted the value of using conditional logic (ifelse()) to efficiently transform variables in R.

Task 5 State-Level Frequency Analysis

A frequency table was produced for the State.UnionTerritory column to examine how reporting was distributed across Indian states and union territories:

Data quality note: The frequency table revealed inconsistencies in state naming — for example, duplicate entries caused by trailing spaces or alternate spellings of the same territory. This highlighted the need for a data cleaning step (e.g., trimws() or manual standardisation) before relying on state-level counts for analysis.
Examining state-level reporting frequencies highlighted the uneven distribution of entries across regions. Some states, such as Kerala and Delhi, had far more records than others. This task emphasised the need to consider data collection bias, reporting practices, and even spelling inconsistencies (e.g., duplicate category names) when working with real-world datasets. It strengthened my awareness of data cleaning as a critical step in analysis.

Overall Reflection

Unit 3 brought together a sequence of related transformation techniques — binary indicators, labelled factors, and banded categories — and showed how each builds on the last to produce progressively more interpretable data. The shift from a raw Deaths count to has_deaths (numeric binary) to has_deaths_factor (labelled factor) is a microcosm of the broader data preparation pipeline: raw values are rarely analysis-ready, and the intermediate representations we create determine the quality and clarity of what follows.

The cut() function introduced a different flavour of transformation — discretising a continuous variable into ordered categories. Choosing appropriate breakpoints is a non-trivial decision that requires domain knowledge; the thresholds used here (0, 5, 15, 16+) reflect epidemiological intuition about what constitutes a meaningful difference in case load at the state level in early 2020.

The state-level frequency analysis was the most practically instructive task, precisely because it surfaced a real data quality issue: inconsistent naming of the same territories. Encountering this in a relatively small dataset underscored how pervasive such problems are in real-world data pipelines, and why data profiling and cleaning must precede any analytical conclusions. Overall, this unit reinforced that the rigour of a data analysis is determined as much by how carefully the data is prepared as by the sophistication of the methods applied to it.

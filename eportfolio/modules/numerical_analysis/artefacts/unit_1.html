Data Activity 1

Dataset

COVID-19 India Cases (January 2020 – March 2020)
This dataset contains early COVID-19 case data from India covering the initial months of the pandemic from January to March 2020. It includes confirmed cases (both Indian and foreign nationals), recovered cases, and deaths across different states and union territories of India.

Task 1.1 — Basic Dataset Information

The dataset contains 7 variables and 167 observations. The variables include 4 numeric columns (confirmed cases for Indian nationals, foreign nationals, cured cases, and deaths), 2 character columns (State/Union Territory and status notes), and 1 date column (Date, after explicit conversion from character).

Task 1.2 — Data-Related Information

  • Unique states / union territories: 23
  • Date range: 30 January 2020 to 29 March 2020
  • Most frequently appearing state: Kerala

This activity introduced me to the practical workflow of loading, inspecting, and querying a real-world dataset in R — skills that sit at the foundation of any data science pipeline. Working with a COVID-19 dataset from the early months of the pandemic added a meaningful context: the numbers were not abstract; they represented the spread of a disease across Indian states at a time when very little was known about it.

The most instructive moment was discovering that R does not automatically parse date columns when reading a CSV file. The initial sapply(data, inherits, "Date") check returned 0 date columns, which was unexpected. This required an explicit conversion using as.Date() with the correct format string before any date-based operations — such as computing the range — could be performed reliably. This reinforced an important principle: understanding how data is stored, not just what data is present, is critical to avoiding silent errors in analysis.

I also encountered a column naming issue: the column referenced in the instructions as State/UnionTerritory was stored by R as State.UnionTerritory (with a period), because R replaces special characters in column names by default. Adjusting the code to match the actual column name highlighted the importance of always inspecting variable names with names() before attempting to reference them.

Overall, this activity provided a systematic, hands-on introduction to the key concepts of this unit: identifying data types, understanding data structure, and distinguishing between numeric, categorical, and temporal variables. These are precisely the foundational statistical and mathematical concepts the module aims to build — specifically, the ability to assess a dataset's composition before applying any analytical technique, ensuring that subsequent inferences rest on a well-understood data foundation.

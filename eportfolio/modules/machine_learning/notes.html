Unit 3 – Correlation and Regression

Lecturecast Notes

Correlation

  • Correlation aims to find the degree of relationship between two variables x and y.
  • Best visualisation method: scatter plot.
  • Correlation results can be: positive (both variables change in the same direction), negative (variables change in opposite directions), or neutral (no relationship).
  • Correlation ≠ Causation, although both may exist at the same time.

Quantifying Correlation

  • Covariance — a measure of how two random variables change together.
    • Positive sign (+): variables move in the same direction.
    • Negative sign (−): variables move in opposite directions.
    • Large magnitude indicates strong correlation.
    • Covariance is dependent on the variability in the data.
  • Pearson Correlation Coefficient (r) — only measures linear relationships.
    • Limitations: sensitive to outliers; cannot be used to predict anything but a linear relationship.

Regression

  • Multiple Linear Regression: used to predict the correlation between more than one independent variable and a dependent variable.
  • Regression can be used for classification.
  • Logistic Regression: predicts a binary outcome (yes/no) based on a set of independent variables.
  • Decision tree type classifying algorithms: used for definitive multi-class classification.

Unit 5 – Clustering

Lecturecast Notes

What is Clustering?

  • Unsupervised machine learning technique.
  • A set of techniques used to partition data into groups (clusters).
  • Clusters are loosely defined as groups of data objects that are more similar to other objects in their cluster than to objects in other clusters.
  • Clustering helps identify two qualities of data: meaningfulness (clusters expand domain knowledge) and usefulness (clusters can serve as an intermediate step in a data pipeline).

Applications of Clustering

  • Retail – cluster customers according to buying patterns.
  • Banking – identification of genuine customers (e.g. fraud detection in loans).
  • Insurance – mostly fraud detection and identification of potential risky customers.

Segmentation

  • Segmentation: technique to divide a population into logical groups with a common need.
  • Base: used for defining clusters.
  • Descriptor: used to profile and define identified clusters.

Similarity and Dissimilarity Measures

Measured by distance between data objects using:

  • Euclidean Distance
  • Simple Matching Coefficient (SMC)
  • Jaccard Coefficient
  • Cosine Similarity
  • Pearson Correlation Coefficient
  • Bregman Divergence

Types of Clusters

  • Well-separated: any point in a cluster is closer to every other point in the cluster than to any point outside it.
  • Centre-based: objects are closer to their cluster centre than to the centre of any other cluster.
  • Density-based: a dense region of data points separated by low-density regions.
  • Concept-based: data share a common property or concept.

Clustering Methods

  • K-Means: Partitioning Method
  • Hierarchical clustering – Agglomerative Method
  • DBSCAN: Density-Based Spatial Clustering of Applications with Noise

K-Means Clustering

Assumes a fixed number of clusters K, with a goal to create "compact" clusters.

Steps:

  1. Randomly select k centroids (data points representing cluster centres).
  2. Apply a two-step expectation–maximisation process:
    • Expectation step: assigns each data point to its nearest centroid.
    • Maximisation step: computes the mean of all points in each cluster and sets the new centroid.

Strengths: simple iterative method; the user provides K.

Weaknesses: sometimes too simple (yields poor results); difficult to guess the correct K.

Choosing an Appropriate K Value

  • The Elbow Method: run K-Means multiple times, incrementing K on each iteration, and record the SSE.
  • The Silhouette Coefficient: quantifies how well a data point fits its assigned cluster by measuring how close it is to other points in its cluster, and how far it is from points in other clusters.

Cluster Evaluation

  • Determine cluster tendency.
  • Determine the correct number of clusters.
  • Evaluate how well clusters fit the data.
  • Compare clusters to externally known results.
  • Compare two sets of clusters to determine which is better.

Cohesion → measures cluster homogeneity. Separation → measures how well separated clusters are.

A good clustering produces clusters with low cohesion value (data points close together / homogeneous) and high inter-cluster separation.

Hierarchical Clustering

  • Agglomerative Clustering: start with each data item as a cluster, then iteratively merge the closest pair of clusters.
  • Divisive Clustering: start with one all-inclusive cluster, then iteratively split it until only singleton clusters remain.

Unit 8 – Training an Artificial Neural Network

Lecturecast Notes

Reducing the Error

  • For a neural network to learn, weights associated with neuron connections must be updated after forward passes of data through the network.
  • These weights are adjusted to reconcile the differences between the actual and predicted outcomes for subsequent forward passes.
  • We don't use fixed step gradient descent — we may miss the local minima.

Backpropagation

  • A mechanism used to update the weights using gradient descent.
  • It calculates the gradient of the error function with respect to the neural network's weights.
  • The calculation proceeds backwards through the network.

Training Goals

  • The main goal of training is to reduce the error (the difference between prediction and actual output).
  • Actual output is "not changing" = constant → so we change the prediction value to reduce error.
  • Error = difference between actual output and predicted output at each data point.

Gradient Descent

  • An iterative optimisation algorithm used to find the minimum of a function.
  • Derivatives or slope changes enable us to quantify change.
  • For adjusting weights within the network, we use this gradient approach — using the derivatives of the network's weights with respect to the output error.

Delta Rule (used in backpropagation)

  • Output Layer Delta: Δoutput = error × sigmoidderivative
  • Hidden Layer Delta: Δhidden = sigmoidderivative × Weight × Δoutput

Bias

Bias in a neural network can be thought of as analogous to the role of a constant in a linear function: the line is effectively transposed by the constant value (no gradient change).

Unit 9 – Introduction to Convolutional Neural Networks

Lecturecast Notes

What is a CNN?

  • Convolutional Neural Network (ConvNet/CNN): a Deep Learning algorithm.
  • Input → Image → Assign importance (learnable weights and biases) to various aspects/objects in the image, enabling differentiation between them.
  • Result: objects within the image can be classified by ANN.
  • Detection + Classification = CNN's power.
  • Images are represented as matrices of pixels, each with a numerical value.

Goal

To reduce images into a form easier to process without losing features that are critical for classification.

Two Main Steps

  1. Feature Learning
  2. Classification

Three Sub-Processes

  • Filter: scans and converts the area covered by the kernel into a weighted sum of pixel values according to settings.
  • Detect: once converted, uses an activation function like ReLU to detect important features and discard unimportant ones.
  • Condense: features are enhanced by pooling, which takes the maximum value of the feature — condensing the image by retaining the highest value.

Filtering

  • We apply certain weight matrices known as kernels.
  • Kernels act like a lens — scanning the image and producing a weighted sum of pixel values.
  • During training, a CNN learns what features it needs to solve the classification problem — i.e. finding the best values for its kernels.

Detection

  • Feature maps are passed through the ReLU activation function (a nonlinear function that separates a particular layer from the convolution layer).
  • Negative values are unimportant and set to 0.

Condensation / Pooling

  • Responsible for reducing the spatial size of the Convolved Feature.
  • Done by either taking the max or average value of the portion of the detected feature map covered by the kernel.

Other Concepts

  • Epoch: the number of passes of the entire training dataset.
  • Softmax Classification technique: classifies the dominating and low-level features distinguished by the model.

Unit 11 – Model Selection and Evaluation

Lecturecast Notes

Overfitting vs Underfitting

  • Overfitting: when a model does well on training data but not on test data.
  • Underfitting: when a model performs poorly on both test and training data.

Sources of Predictive Error

  • Statistical noise in the data
  • Incompleteness of the data sample
  • Limitations of each different model type

Bias vs Variance

  • Bias: high bias can cause an algorithm to miss relevant relations between features and target outputs (underfitting).
  • Variance: error from sensitivity to small fluctuations in the training set (overfitting).

Reducing Underfitting

  • Bigger network
  • Train for a longer duration
  • Increase the number of parameters in the model

Reducing Overfitting

  • Add more data
  • Data augmentation
  • Simplify the model
  • Change the training process
  • Early termination
  • Regularisation (Dropout, DropConnect, L1 and L2 regularisation)

Ideas to Improve ML/DL Strategies

  • Collect more data
  • Collect more diverse training examples
  • Train algorithm longer with a suitable optimiser
  • Try bigger or smaller networks
  • Try dropout
  • Add regularisation

Model Selection Techniques

  • Probabilistic Measures: choose a model via in-sample error and complexity. Appropriate for simpler linear models (e.g. linear regression) where the calculation of model complexity penalty is known and tractable.
  • Resampling Methods: choose a model via estimated out-of-sample error — splitting the training dataset into sub-train and test sets, fitting a model on the sub-train, and evaluating it on the test set.

Cross Validation

Split the dataset into two sets: training and validation. For each candidate model, form a classifier using the training data and apply it to the validation set. The final classifier is the one that classified the highest fraction of inputs correctly.

Performance Metrics

  • Sensitivity = True Positive rate
  • Specificity = True Negative rate
  • Sensitivity and specificity are negatively correlated: if sensitivity increases, specificity decreases.

Type I and Type II Errors (pregnancy example)

  • Type I: classify a man as pregnant.
  • Type II: classify a pregnant woman as non-pregnant.

ROC Curve and AUC

  • Receiver Operating Characteristics (ROC) Curve and Area Under the Curve (AUC).
  • For measuring the relationship between sensitivity and specificity, the ROC curve is used — the AUC provides a measurement ratio value between them.
  • Moving the threshold left (lower value) → sensitivity increases (more True Positives correctly identified), but specificity decreases (more True Negatives classified as positive → higher False Positive rate).
  • Moving the threshold right (higher value) → the opposite happens.
